Apa Itu Flowchart?

Flowchart (diagram alur) adalah representasi visual dari algoritma menggunakan simbol-simbol grafis yang dihubungkan dengan panah untuk menunjukkan urutan proses.

🎯 Mengapa Belajar Flowchart?

  • ✅ Memudahkan pemahaman alur logika program
  • ✅ Membantu komunikasi antara programmer dan non-programmer
  • ✅ Mempermudah deteksi kesalahan (debugging) sebelum coding
  • ✅ Standar dokumentasi dalam pengembangan software

Analogi: Peta Perjalanan

Flowchart seperti peta GPS untuk algoritma:

  • 📍 Simbol = Lokasi/titik penting (mulai, keputusan, akhir)
  • ➡️ Panah = Jalan/rute yang harus dilalui
  • 🔄 Percabangan = Pilihan belok kiri/kanan

Dengan peta yang jelas, kita tidak akan tersesat dalam logika program!

Simbol-Simbol Flowchart Standar

Berikut simbol-simbol utama yang wajib kamu ketahui:

START/END

Terminator

Menandai awal atau akhir algoritma

PROSES

Process

Operasi/perhitungan: hitung, assign, ubah data

YA/TIDAK

Decision

Percabangan: keputusan berdasarkan kondisi (Ya/Tidak)

INPUT/OUTPUT

Input/Output

Membaca data atau menampilkan hasil

A

Connector

Menghubungkan bagian flowchart yang terpisah

FLOW LINE

Flow Line

Panah arah alur proses

📐 Aturan Menggambar Flowchart

  • 🔹 Alur dibaca dari atas ke bawah atau kiri ke kanan
  • 🔹 Setiap simbol dihubungkan dengan panah, bukan garis biasa
  • 🔹 Simbol Decision memiliki minimal 2 keluaran (Ya/Tidak, True/False)
  • 🔹 Hindari garis yang saling bersilangan; gunakan Connector jika perlu
  • 🔹 Gunakan kata kerja singkat dan jelas di dalam simbol

Pseudocode: Bahasa Semi-Formal

Pseudocode adalah cara menuliskan algoritma menggunakan campuran bahasa manusia dan struktur pemrograman, tanpa terikat syntax bahasa tertentu.

📝 Struktur Dasar Pseudocode

Komponen Penjelasan Contoh
DEKLARASI Menyiapkan variabel yang akan digunakan nama, umur : string
INPUT Membaca data dari pengguna Baca(nama)
PROSES Operasi/perhitungan data total ← harga × jumlah
OUTPUT Menampilkan hasil ke pengguna Tulis("Halo, ", nama)
KONDISI Percabangan logika Jika umur ≥ 17 maka ...
PERULANGAN Mengulang proses Untuk i ← 1 sampai 10 lakukan ...

Contoh: Menghitung Luas Persegi Panjang

ALGORITMA HitungLuasPersegiPanjang // Deklarasi variabel DEKLARASI: panjang, lebar, luas : real DESKRIPSI: // Input Baca(panjang) Baca(lebar) // Proses luas ← panjang × lebar // Output Tulis("Luas persegi panjang: ", luas) SELESAI

💡 Tips Menulis Pseudocode yang Baik

  • Gunakan kata kunci konsisten: Baca, Tulis, Jika, Untuk, Mientras
  • Tulis satu perintah per baris agar mudah dibaca
  • Beri komentar (//) untuk penjelasan kompleks
  • Gunakan indentasi untuk blok kode (If, For, While)
  • Hindari syntax spesifik bahasa pemrograman (tidak ada ;, {}, dll)

Konversi: Dari Ide ke Representasi

Mari praktikkan konversi algoritma sederhana ke dalam 3 bentuk: deskripsi, flowchart, dan pseudocode.

🎯 Studi Kasus: Menentukan Bilangan Ganjil/Genap

Deskripsi Algoritma

  1. Mulai
  2. Baca sebuah bilangan bulat n
  3. Hitung sisa bagi n dengan 2
  4. Jika sisa = 0, maka bilangan genap
  5. Jika sisa ≠ 0, maka bilangan ganjil
  6. Tampilkan hasil
  7. Selesai
